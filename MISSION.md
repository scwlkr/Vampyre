# Vampyre Mission

***Vampyre is a system for creating, sustaining, and improving software projects from one always-on runtime host.***

---

## Vampyre System Dream

>| This is the future version of Vampyre. Not the dry spec. The feeling.

### Builder Mode

```txt
I am nowhere near my computer when the idea hits.

I am sitting on the toilet, half distracted, and suddenly I know exactly what I want: a tiny app that does x in a way that feels sharp and weirdly useful.

So I open Telegram and message Vampyre:

/newProject "project x short description"

A few minutes later Vampyre sends back 2 possible versions.

One is small and practical. The other is more ambitious and interesting. It gives me the tradeoffs, the rough shape of each app, and what it would build first.

I pick the second one.

Vampyre creates the repo, writes the first project docs, sketches the roadmap, and starts building. When it hits something only I should decide, it opens a GitHub approval and sends me the link in Telegram.

I approve one thing. I reject another. Then I forget about it and go live my life.

Later that night Vampyre sends me a check-in. The app boots. The first feature works. Tests are passing. It includes a screenshot, a short list of what changed, and one question about the next fork in the road.

In the morning, I wake up and the project feels real.

Not like a note in my phone. Not like another idea I abandoned.

There is a repo. There is an app. There is a status. There is a next step.

I open it, play with it, and say:

"wow, this is actually good. add x."

Vampyre adds x, proves it, and sends me the result.

At some point the project stops feeling like a build experiment and starts feeling like something I want to keep. So I tell Vampyre to move it from Builder Mode into Watcher Mode.
```

---

### Watcher Mode

```txt
Now the project is alive and useful, but I do not want it rotting in a folder somewhere.

Watcher Mode keeps an eye on it. Vampyre checks the repo, the open issues, the status docs, the failing tests, the weird stale branches, and the little rough edges I would normally forget about.

One morning I wake up and Vampyre has sent me a short Telegram brief. It says the project is healthy, but there are 3 things worth doing: one tiny bug, one cleanup that will make future work easier, and one feature idea that came from how the app is actually shaping up.

I am still half asleep, so I just tap the GitHub approval for the tiny bug and ignore the rest.

Vampyre makes a worktree, fixes the bug, runs the tests, opens a PR, and sends me the link. I review it when I have time. Vampyre does not need me to babysit every command.

Weeks go by like this. The project does not explode into random churn. It slowly gets better. Bugs get handled, docs stay close to the truth, small features keep landing, and the project keeps feeling cared for even when I am busy with something else.
```
